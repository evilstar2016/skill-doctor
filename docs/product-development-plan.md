# Skill Doctor 研发交付计划

最后更新：2026-07-23  
目标：将 Skill Doctor 演进为可在本地完整运行、可管理多种 Agent 资产、并可通过标准 API 调用云端或本地模型服务的产品版本。

## 交付原则

- 本地优先：扫描、资源管理和变更执行由本地 UI 服务完成。
- 标准模型接口：优先接入 OpenAI-compatible HTTP API；云端公开模型与本地推理服务使用同一套 endpoint、model、apiKey 配置。
- 不下载模型：本阶段不安装模型文件、不启动或运维推理引擎；本地模型仅指向用户已经运行的兼容服务（例如本机或局域网 endpoint）。
- 完整业务闭环：每个功能必须具备配置、后端持久化/执行、前端状态呈现和自动化验证，不以孤立演示页交付。
- 当前阶段暂不扩展协议治理、审批门禁与安全产品能力；保留现有必要的本地文件保护和确认机制。

## 模型配置示例

云端与本地都使用同一标准格式：

```json
{
  "analysis": {
    "baseUrl": "https://api.openai.com/v1",
    "model": "gpt-4.1-mini",
    "apiKey": "your-api-key",
    "timeoutMs": 30000
  },
  "embedding": {
    "baseUrl": "http://127.0.0.1:11434/v1",
    "model": "nomic-embed-text",
    "apiKey": "optional"
  }
}
```

`baseUrl` 指向任意实现 OpenAI-compatible `/chat/completions` 或 `/embeddings` 的服务；Skill Doctor 不负责下载模型或启动这些服务。
