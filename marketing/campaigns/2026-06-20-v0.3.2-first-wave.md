## English Short Post

I shipped `skill-doctor` v0.3.2: a local audit CLI for AI agent skills, rules, and instruction files.

If your Claude Code, Cursor, Copilot, Codex, or Gemini CLI setup has accumulated config from different tools and guides, it can flag:

- duplicate skills
- overlapping triggers
- risky instructions such as token exposure, shell execution, or data upload wording

## HN / Reddit Version

Title options:

- Show HN: skill-doctor, a local audit CLI for AI agent skills
- A local audit CLI for Claude Code skills, Cursor rules, Copilot instructions, and Codex AGENTS.md
- AI agent skills are becoming dependency trees, so I made an audit tool for them

Post:

AI coding agents now have their own configuration layer: Claude Code skills, Cursor rules, Copilot instructions, Codex AGENTS.md, Gemini CLI skills, Windsurf rules, and more.

After enough installs and copied guides, the agent can behave inconsistently because two skills trigger on the same task, or because a risky instruction is buried in a file nobody reviewed.

I built `skill-doctor` to inspect that layer locally. It detects duplicate skills, overlapping triggers, suspicious instructions, cleanup candidates, and cross-agent drift. It supports Claude Code, Cursor, GitHub Copilot, Codex, Gemini CLI, Windsurf, Kiro, Trae, OpenCode, OpenClaw, and Hermes.

Run it:

```bash
npx @evilstar2025/skill-doctor scan
```

Or try the included safe demo:

```bash
git clone https://github.com/evilstar2016/skill-doctor.git
cd skill-doctor/examples/conflicted-agent-project
npx @evilstar2025/skill-doctor scan --scope project
npx @evilstar2025/skill-doctor conflicts --scope project
npx @evilstar2025/skill-doctor audit --scope project
```

The safety boundary: it reads local files and does not upload skills by default.
