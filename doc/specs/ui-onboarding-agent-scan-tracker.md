# UI 首次体检与多 Agent 体验开发跟踪

目标：让新用户无需阅读命令文档，即可确认检查目录、选择本机正在使用的 Agent，并理解体检深度与共享资源影响范围。

更新规则：每完成并验证一项功能，立即将对应 checkbox 改为 `[x]`，并在“验证记录”中补充证据。

## A. 首次使用与检查目录

- [x] 首次打开且没有快照时停止自动扫描，展示首次配置页。
- [x] 展示默认项目目录，并说明它用于发现项目级 Agent 配置。
- [x] 支持输入其他项目目录；服务端校验目录存在且确实为目录。
- [x] 清楚区分“项目配置”和“用户全局配置”，不引入额外的“工作路径”概念。
- [x] 完成首次配置后保存项目目录和扫描偏好，再开始第一次体检。
- [x] 已有快照或已完成首次配置时不重复打断用户。

## B. Agent 自动发现与选择

- [x] 启动时执行轻量探测，返回当前项目或用户目录中实际存在配置的 Agent。
- [x] 区分“当前项目已使用”和“仅发现全局配置”，优先推荐前者。
- [x] 首次配置页默认选中推荐 Agent；没有可靠推荐时选择“全部”。
- [x] 页面顶部增加独立 Agent 选择栏，支持“全部”或单个 Agent，并可重新体检。
- [x] Agent 数量较多时允许横向滚动，不挤压主操作区。
- [x] Agent 选择按项目持久化；首次推荐不在每次启动时重复弹出。

## C. 分析模式与说明

- [x] 首次配置页和顶部显眼位置提供“标准体检 / 深度体检 / 自定义”。
- [x] 标准体检启用静态检查、关键词冲突和上下文成本，不启动 MCP server 或外部分析服务。
- [x] 深度体检根据已配置能力启用语义冲突、AI 审计和冲突边界分析。
- [x] 深度能力不可用时展示原因和配置提示，不显示成可执行的正常选项。
- [x] 将 MCP 工具发现明确标记为可能启动本地 server 的独立能力。
- [x] Tokenizer、插件缓存、上下文成本等术语提供可点击、可聚焦的说明，不只依赖鼠标悬停。
- [x] 高级预算和 tokenizer 参数继续保留在自定义设置中。

## D. 多 Agent 共享资源

- [x] 后端以规范路径识别同一物理资源，并保留各 Agent 的独立加载关系。
- [x] 共享资源模型包含使用它的 Agent、范围、启用状态和成本语义。
- [x] 共享 Skill 不因被多个 Agent 引用而误报为重复安装。
- [x] 资源清单为共享 Skill/MCP 显示“共享”标识和 Agent 数量。
- [x] 资源详情展示全部使用方，并说明修改会影响哪些 Agent。
- [x] Agent 筛选下仍能找到共享资源，并保留正确的问题关联。
